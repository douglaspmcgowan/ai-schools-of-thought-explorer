## Techniques & tricks

**Context engineering is replacing prompt engineering** ([dev.to/gabrielhca/context-engineering-the-skill-replacing-prompt-engineering-in-2026-3lgd](https://dev.to/gabrielhca/context-engineering-the-skill-replacing-prompt-engineering-in-2026-3lgd)): The highest-leverage LLM skill is now designing the information architecture around a prompt — what goes in the context window, in what order, how much, and how it's structured — rather than wordsmithing the instruction itself; the concrete result is production outputs that hold up at scale where clever phrasing doesn't.

**"Think step by step" now hurts reasoning models** ([the-ai-corner.com/p/context-engineering-guide-2026](https://www.the-ai-corner.com/p/context-engineering-guide-2026)): Explicit chain-of-thought prompts like "think step by step" measurably degrade performance on GPT-5, Claude 4+, and similar reasoning-native models that handle CoT internally — removing the instruction is the concrete fix.

**GitAgentProtocol (GAP) — git commits as agent audit trail** ([gitagent.sh](https://www.gitagent.sh/)): Every agent decision, memory write, and rule change is stored as a git commit, giving you an exact, diffable audit trail of what the agent did and why without adding extra observability tooling; switch between Claude, OpenAI, or Gemini by changing one line.
