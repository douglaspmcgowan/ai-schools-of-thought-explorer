## Techniques & tricks

**System prompt as version-controlled config** ([https://www.premieritsolutions.com/blogs/prompt-engineering-for-ai-agents/](https://www.premieritsolutions.com/blogs/prompt-engineering-for-ai-agents/)): Treat the agent system prompt like production configuration code—define identity, tools, decision logic, and escalation paths, check it into git, and write tests for it—producing dramatically more consistent behaviour in multi-agent pipelines.

**Compound Engineering loop** ([https://every.to/chain-of-thought/compound-engineering-how-every-codes-with-agents](https://every.to/chain-of-thought/compound-engineering-how-every-codes-with-agents)): Orchestrate agents in a Plan→Work→Review→Compound cycle where each iteration's lessons feed back as improved context, so development velocity increases rather than plateaus over time.

**Pelican-bicycle SVG test for unbiased model comparison** ([https://simonwillison.net/2026/may/19/5-minute-llms/](https://simonwillison.net/2026/may/19/5-minute-llms/)): Prompt any model to generate an SVG of a pelican riding a bicycle—too absurd for any lab to train toward, never in benchmarks—and get an instant signal on spatial reasoning and instruction-following capability differences between models.
