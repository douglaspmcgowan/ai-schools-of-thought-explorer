## Techniques & tricks

**Autonomous Self-Refinement with Numerical Targets** (https://lushbinary.com/blog/advanced-prompt-engineering-techniques-developer-guide/): Instead of asking for a score, instruct the model to "iterate against your own rubric until you reach 9.5 across all criteria, then show me the final version" — the model self-grades, revises, and reruns 5–6 rounds in one response, producing measurably sharper output than a single-pass review.

**Instruction-First Role Definition** (https://dev.to/uenyioha/prompting-techniques-that-actually-work-lessons-from-automating-architecture-analysis-57al): Open with an explicit role + abstraction constraint ("You are an architecture analysis agent. Output C4 container-level abstraction unless explicitly requested otherwise") before any task description — practitioners report this immediately stops generic summaries and produces on-target structured analysis.

**Reproducible Context Assembly Pipeline** (https://bits-bytes-nn.github.io/insights/agentic-ai/2026/04/05/evolution-of-ai-agentic-patterns-en.html): Replace ad-hoc string concatenation with named, ordered stages — inject system prompt → select relevant history → summarize tool results → check token budget — making agent context predictable, inspectable, and debuggable across runs.
