## Techniques & tricks

**Agentic Engineering Patterns — 12-chapter living guide** (https://simonwillison.net/2026/Feb/23/agentic-engineering-patterns/): Simon Willison's Gang-of-Four-style guide for coding-agent workflows, built around two foundational principles: "code is now inexpensive" (shift focus from typing to architecture) and "preserve domain expertise" (keep judgment work on the human side).

**Specification-over-prompting with Ouroboros** (https://github.com/Q00/ouroboros): Running a structured Socratic interview to crystallize requirements into a verified spec before any code is generated produces consistently correct outputs without iterative prompt tuning.

**Ollama Modelfiles as callable specialists** (https://www.kunalganglani.com/blog/running-local-llms-2026-hardware-setup-guide): Baking a system prompt, temperature setting, and context-window size into a named Modelfile lets you call a fully configured specialist model with a single `ollama run mymodel` — the LocalLLaMA community's most-cited workflow optimization.

**Ebbinghaus decay for agent memory management** (https://github.com/sachitrafa/YourMemory): Applying the biological forgetting curve (decay rate modulated by retrieval frequency × importance score) to memory pruning nearly doubles recall versus naive context truncation and prevents "context rot" in long-running agents.

**AI-DLC phase-gated workflow steering** (https://github.com/awslabs/aidlc-workflows): Layering YAML steering rules over any coding agent enforces explicit human-approval checkpoints between phases and skips stages that add no value for the current request — works as an overlay on top of existing agent tools.
