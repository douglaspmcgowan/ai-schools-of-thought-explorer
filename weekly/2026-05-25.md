## Techniques & tricks

**Context-fork skill isolation** (https://github.com/shanraisshan/claude-code-best-practice): Declaring `context: fork` in a Claude Code skill keeps all intermediate tool calls — file reads, searches, experiments — inside the child agent's context rather than polluting the parent, producing cleaner state for the main session.

**Session context budget rule** (https://github.com/shanraisshan/claude-code-best-practice): Cap sessions at 40% context usage and hard-reset at 60%; always supply a descriptive hint when calling `/compact` rather than letting it auto-compact — results in dramatically more coherent long-session continuations.

**Test-time compute via parallel context windows** (https://github.com/shanraisshan/claude-code-best-practice): Running a second, fresh Claude instance as a code reviewer (via git worktrees) finds more bugs than asking the writing instance to self-review — separating generation and review context windows exploits the model's variance.

**Spec-driven development workflow** (https://github.com/github/spec-kit): Write a human-readable spec first with `/speckit.specify`, then hand it to an AI agent with `/speckit.implement` — the spec acts as a contract that prevents agents from silently changing scope and makes partial completions reviewable by design.

**Karpathy's four-rule CLAUDE.md** (https://github.com/multica-ai/andrej-karpathy-skills): Dropping a single CLAUDE.md file encoding four rules (state assumptions, write minimal code, touch only what's required, define verifiable success criteria) consistently reduces speculative over-engineering and silent scope expansion across any model.
