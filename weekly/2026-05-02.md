## Techniques & tricks

**Context packing before coding** (https://addyosmani.com/blog/ai-coding-workflow/): Before prompting, dump all relevant project context — docs, existing patterns, invariants, niche API READMEs — into a single text block using gitingest or repo2txt; this eliminates the LLM "flying blind" problem and produces more consistent, on-pattern output.

**Pattern priming with inline examples** (https://addyosmani.com/blog/ai-coding-workflow/): Paste 1-2 examples of the exact code style or function shape you want before asking the LLM to generate new code — LLMs mimic reliably, locking in naming conventions, error-handling style, and structure without lengthy prose instructions.

**Codex CLI /goal loop** (https://simonwillison.net/2026/Apr/30/codex-goals/): In Codex CLI 0.128.0, set `/goal <objective>` and the agent loops autonomously — re-evaluating its own progress each turn — until it judges the goal achieved or hits your token budget, enabling long-horizon tasks without babysitting.
