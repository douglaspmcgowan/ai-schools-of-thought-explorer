Agent Architecture & Coordination

When you give an LLM tools and let it work, how should the whole thing be shaped? One loop that keeps spinning? A graph of branching states? A crew of specialist agents? A durable workflow that survives crashes? Six answers, each backed by a live framework and a working philosophy.

The shared problem. An agent runs for a long time, spends real money on tokens, calls external tools that can fail, and sometimes produces wrong output. These six schools each optimize for a different part of that problem — simplicity, specialization, state, reliability, or context quality.

Single-loop

One model, one context window, one main loop. Let the agent read, call tools, and write until it's done. Claude Code is the canonical example.
