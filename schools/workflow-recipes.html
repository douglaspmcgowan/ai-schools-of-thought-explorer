Workflow recipes

End-to-end ways of working. Not a prompt, not a tool — a whole recipe. Agentic engineering, Claude Code for designers, sketching in code: repeatable loops that combine prompts, tools, and habits into something portable.

Recipes vs. prompts. A prompt is a line or two. A recipe is a full workflow — what you set up first, how you break the task, what you check at each stage. The voices here write at the recipe altitude.

The argument

Addy Osmani's Agentic Engineering is the most influential recipe essay of this cycle — the name "agentic engineering" itself comes from here, and the piece is structured as a set of reusable patterns rather than a single project walkthrough. Pairs with his 80% Problem for the honest accounting of where the recipes break.

For designer-specific recipes, ADPList's Claude Code Guide for Designers is the clearest onboarding. For a different kind of creative recipe, Sketching with Code (Proof of Concept) treats code as a sketching medium — the AI agent as part of an iterative visual-thinking loop, not as a code generator.

The web-developer-side recipe is Leveraging OpenClaw as a Web Developer — specifically how one practitioner has integrated the OpenClaw agents into day-to-day web work. Less polished than Osmani but more concrete about which parts of a real workflow each agent owns.
