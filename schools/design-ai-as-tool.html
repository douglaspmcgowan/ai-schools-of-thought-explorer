AI as design tool

The hands-on school — image and code models sit inside the design surface and do actual design work. Less theory, more "here's the prompt, here's the result."

The useful test. Can a designer do a real job faster with the model than without it? This school is populated by writers who answer that question with screenshots, not essays.

The argument

The clearest recent demonstration is raw.studio's UI Design with Nano Banana Pro — a practical walkthrough of using an image model as a working UI-generation tool, with the specific prompts and the resulting artifacts. The piece's value is that it's a recipe, not a prediction.

The code-side analogue is Figma's announcement of Claude Code inside Figma (see also the new applications entry). Notable as a mainstream design tool adopting an agent-native workflow in-surface rather than asking designers to leave Figma for a separate chat window. ADPList's Claude Code Guide for Designers is the practitioner-side take — the same school as the Figma integration, but from the designer's desk.
