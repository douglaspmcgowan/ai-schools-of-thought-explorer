Agent internals & cost curves

What agents actually do at runtime. Visualizing the reasoning chain, dissecting specific commands, and pricing out the quadratic cost of long contexts. Useful before you take any architectural claim on faith.

Anchor the debates. Many agent-architecture claims rest on unexamined assumptions about what a reasoning step costs, how long it takes, and what the model is doing during it. These pieces make those assumptions visible.

The argument

Render-of-Thought: Visualizing Reasoning Chains (arXiv, 2026) is the interpretability-adjacent paper on actually seeing what a reasoning model does step-by-step. Beyond research value, it's practically useful for debugging long agent runs — the kind of thing you reach for when an agent's output looks fine but was clearly reached by a weird path.

For a specific agent's innards, Zolkos' Deep Dive: How Claude Code's /insights Command Works reverse-engineers a single Claude Code feature to show how a serious production agent structures its own diagnostics. It complements Cracking The Claw from CTO Lunch NYC, which does the same kind of dissection at a higher level.

On the cost side, Expensively Quadratic: the LLM Agent Cost Curve is the piece to send anyone excited about long-context agents — the quadratic term is real, and it reshapes which architectures are actually affordable. The Economics of LLM Inference (mlechner) is the batched-serving analogue — why inference costs what it does at the provider level.
