Autonomous agent

Hand the AI a repo, a shell, and a goal. It reads, plans, writes, runs tests, commits. You review the diff when it comes back. The human is still in the loop — but at review-time rather than keystroke-time.

The shift. "AI coding" used to mean autocomplete. The autonomous-agent school reframes it: the human writes the goal, not the code. The skill that matters is code review plus scope setting.

The argument

Claude Code, Devin, Jules, Codex, and Amp are the production examples. The practitioner case for the category is made most cleanly in "Why Developers Keep Choosing Claude Over Every Other AI" — the argument being that autonomous agents finally stopped feeling like supervised interns and started feeling like engineers you trust to come back with a finished diff.

The proof of concept for the upper bound is OpenAI's documented 25-hour GPT-5 Codex sprint: a single agent holding coherent state over a full day of coding. The Eng Leadership piece on the Codex team's workflow is the counterweight to the Anthropic-centric narrative — useful for seeing how a second leading lab thinks about the same problem.

The school's most serious internal critic is Addy Osmani's "80% Problem": autonomous agents stall predictably at ~80% completion on non-trivial tasks. Osmani's recommended fix leads into the spec-driven school. A harsher critique comes from Christopher Zerafa, "Agentic Coding Is a Dead End" — worth reading as the strongest public counter-position.
