Single-loop

One model. One context window. One long main loop that reads, calls tools, and writes until the work is done. No orchestration layer, no graph, no crew — the architecture is the while-loop.

The load-bearing claim. If the model is good enough and the context is managed well, additional structure outside the loop adds more failure modes than it prevents. You don't need a graph; you need a better model and a cleaner prompt.

The argument

Claude Code is the archetype of this school, and the case for it has been made most cleanly by observers rather than its makers. Simon Willison, citing Chris Lloyd, points out that Claude Code is "just a TUI" — the magic isn't in hidden orchestration, it's in how thin the harness around the model actually is. Amp, shipping a directly competing tool, arrived at the same conclusion and titled the essay "The Coding Agent Is Dead": the standalone agent-as-product is collapsing into the single-loop shape because raw model capability plus a clean loop beats the orchestration games.

A practitioner's view of why this school won is in Manish Bhusal's "Why Developers Keep Choosing Claude Over Every Other AI": taste in tool use, restraint, and willingness to re-read its own work — traits that benefit from a single coherent context rather than a multi-agent relay race. The GPT-5 Codex 25-hour sprint is the existence proof that a single-loop agent can hold together over very long horizons if the context is curated well.

Adjacent tools — OpenClaw (leveraging OpenClaw as a web developer), Aider, Amp — all inherit the shape. What separates them is the TUI and the tool permission model, not the control-flow architecture.
