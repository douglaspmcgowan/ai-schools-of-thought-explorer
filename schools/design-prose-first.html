Prose-first interfaces

If the model runs on text, the primary design artifact should be text too. The school argues that prompts, specs, and prose are the real interface — pixels and components come after.

The contrarian part. Traditional design starts from the surface (layout, components, flows). This school flips it: start from the words — what the model should understand, what the user should say — and derive the surface from there.

The argument

The clearest statement is unknownarts' AI Runs on Text. So Should You. The argument: AI systems consume and produce text, so the highest-leverage design work is writing — system prompts, microcopy, spec docs — not pushing pixels. Designers who treat text as the primary artifact ship faster and end up with interfaces that degrade gracefully as the underlying model changes.

Every's Writing With AI is Harder Than You Think (see the Every Misc entry) pairs with this as a corrective. Prose-first doesn't mean "just prompt it" — the craft of writing for and with AI is its own skill, and the piece pushes back on treatments that wave it away.
